inline

Adds new element at the end of vector.
New element is initialized as a copy of value.
If new size is bigger than previous capacity, reallocation takes place.
Reallocates by a factor of 2. (In more modern implementation this factor is 1.5).

Parameters
value- the value of the element to append
Returns
none
Exceptions
exceptionrelated to allocator or element copy/move constructor/assignment.
Remarks
Complexity: Amortized constant //TODO: check why Amortized
If reallocation happens, all references, pointers or iterators related to the underlying data are invalidated
If an error is thrown this function has no effect (= strong exception guarantee)

◆ reserve()

template<class T , class Allocator = std::allocator<T>>
void ft::vector< T, Allocator >::reserve ( size_type  new_cap)
inline

Requests that _capacity should at least contain new_cap elements.
If new_cap is greater than current vector _capacity, function causes container to reallocate increasing capacity to new_cap.
Complexity: if realloc happens, linear in vector_size at most.

Parameters
new_cap- new capacity of the vector, in number of elements
Returns
none
Exceptions
length_error- exception is size requested is greater than vector::max_size
bad_allocator- if reallocation fails due to lack of space (when default alloc is used)
Remarks
Usefull if number of elements is known in advance. Can allocate without instanciation right amount of data, eliminates need to reallocate when vector grows
If an error is thrown by allocation or max_size, leaves vector in his previous state

◆ resize()

template<class T , class Allocator = std::allocator<T>>
void ft::vector< T, Allocator >::resize ( size_type  count,
value_type  val = value_type() 
)
inline

Resizes the container to count elements, does nothing if count == size.
If size > count, container is reduced to its first count elements.
If size < count, additional value elements are appended.

Parameters
count- New size of the container
value- Value to append if size < count
Returns
none
Remarks
Complexity: Linear in the difference between current size and count. Additional Complexity is possible due to reallocation.
Vector capacity is not reduced to avoid invalidating all iterators. Only those affected by pop_back() call are invalidated.
